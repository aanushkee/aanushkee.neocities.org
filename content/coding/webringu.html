Webringu

Webringu is a super simple, easy-to-use and customizable webring script so you can create your own webring today! :) I also use it for my own webring.

The name 'Webringu' is inspired by the movie Ringu (1998) (which is the Japanese original of 'The Ring').

Please link back to this page if you use Webringu for your webring. I appreciate it!

What it does (and doesn't)

  • Webringu is super easy to use. It doesn't require any JavaScript knowledge beyond editing a simple members array.
  • Webringu will work for free Neocities accounts (for both you and your members).
  • Webringu doesn't require your webring's members to include a bloaty script or CSS file on their site.
  • Webringu gives you complete freedom for how the webring widget looks.
  • Webringu provides you with next, previous and random links to use in your widgets.
  • You can edit Webringu's code to your liking if you know what you're doing.

But... Webringu doesn't update your members list automatically. You have to do that manually in the code. However, it can then automatically output the members list on your page. This way, you only have to keep one member list up-to-date (the one in the code).

How To Use

Your Code

First, create a page for your webring which will have all its information, rules, members list, etc. It does not matter whether this is a whole website on its own or just a singular page on your own website.

On that page, use this code:

<script>
  // List of members. You need to update this by hand. Slugs need to be unique and without special characters or spaces.
  const members = [
    {
      url: "https://example0.com/",
      title: "Example0",
      slug: "example0",
    },
    {
      url: "https://example1.com/",
      title: "Example1",
      slug: "example1",
    },
  ];
</script>
<script src="https://petrapixel.neocities.org/assets/js/webringu.js"></script>

If you have a free Neocities account: You won't be able to use an external script. Instead, download it, name it webringu.js, upload it to your website and use it like so: 

<script src="/your-path-here/webringu.js"></script>

The rest of Webringu will work like normal.

You will have noticed that every member has something called a slug. That's just a unique ID that includes no special characters, perferibly written in all lowercase. For example petrapixel. You can let your members choose their own slug or assign one to them yourself - whatever your preference.

You need to keep the members array updated, e.g. when a new member joins, when someone leaves, when someone's URL changes, etc.

Member's Code

Now, assuming your webring page is https://your-webring-page.neocities.org/webring you can give the members of your webring a code that looks something like this:

Your Webring Name
previous random next

This is the code for the widget:

<div class="my-webring-widget">
    <a href="https://your-webring-page.neocities.org/webring"><b>Your Webring Name</b></a><br />
    <a href="https://your-webring-page.neocities.org/webring?slug=YOURSLUGHERE&action=previous">previous</a>
    <a href="https://your-webring-page.neocities.org/webring?slug=YOURSLUGHERE&action=random">random</a>
    <a href="https://your-webring-page.neocities.org/webring?slug=YOURSLUGHERE&action=next">next</a>
</div>

Obviously, add your real webring page URL and name (or image) in there. You can use arrows for the next/previous links (e.g. ⤻ ⤺ → ← » « ⇨ ⇦). You can remove the random link if you don't need it, and you can style the widgets however you want. I recommend simple styling that is achievable with inline CSS only; This way, your members don't have to add a CSS file to their site just to display the widget. But really, you're free to do whatever you want as long as the URLs are correct.

Tell your members to replace YOURSLUGHERE with their slug (unique ID). Otherwise the links will NOT work. To make sure your members know their slug it's a good idea to include it in your members list. (Webringu's automatic output (see below) includes it, of course.)

Optional: Members List

If you want an automatic table of your members outputted on your webring page, just add this element:

<div id="webringulist"></div>
<noscript>Please enable JavaScript to view the member list.</noscript>

In this element, a table with all members will be rendered. It will look something like this:

Member Website Description
Example0 example0.com Example Description
Example1 example1.com Example Description
Example2 example2.com Example Description

The description column is only showed if in use. Slugs are displayed when hovering over a member's name/title.

You can style this table however you want.

To output the number of members in the webring, use <span id="webringucount"></span>, e.g.:
